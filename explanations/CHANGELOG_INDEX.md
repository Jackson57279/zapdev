## How to Read Changelogs

### Understanding the Sections

1. **Added** - New features and functionality
2. **Changed** - Modifications to existing features
3. **Fixed** - Bug fixes and issue resolutions
4. **Security** - Security improvements and vulnerability patches
5. **Deprecated** - Features being phased out
6. **Removed** - Features no longer available
7. **Breaking Changes** - Changes requiring migration

### Using This Documentation

- **For Users**: Check "Added", "Changed", and "Fixed" sections for feature updates
- **For Developers**: Review "Security", "Breaking Changes", and "Migration Guide"
- **For DevOps**: Focus on "Infrastructure", "Deployment", and "Dependencies Updated"
